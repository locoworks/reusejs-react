## ReuseJS Did Mount Effect Hook

### Description

The `useDidMountEffect` is a custom hook that allows you to run an effect only after the component has mounted (similar to the `componentDidMount` lifecycle method in class components). It ensures that the effect function is executed only after the initial render of the component.

Parameters:
- `func` (Function): The effect function to be executed after the component has mounted.
- `deps` (Array): An optional array of dependencies that determines when the effect should re-run. The effect will re-run if any of the dependencies in the array change.

### Usage

1. Import the `useDidMountEffect` hook from the appropriate location.

```jsx
import { useDidMountEffect } from '@locoworks/reusejs-toolkit-react-hook';
```

2. Use the `useDidMountEffect` hook within a functional component.

```jsx
useDidMountEffect(func, deps);
```

- `func` is the effect function to be executed after the component has mounted.
- `deps` is an optional array of dependencies that trigger the effect when they change.

### Example

```jsx
import React from 'react';
import { useDidMountEffect } from '@locoworks/reusejs-toolkit-react-hook';

const MyComponent = () => {
  // Define an effect function
  const handleEffect = () => {
    // Perform some effect operation
    // ...
  };

  // Use the useDidMountEffect hook
  useDidMountEffect(handleEffect, [dependency]);

  // ...
};
```

In this example, the `handleEffect` function will be executed after the component has mounted. If the `dependency` value changes, the effect will not be re-run. This hook is useful for running effects that should only be executed once, specifically after the initial mount of the component.
