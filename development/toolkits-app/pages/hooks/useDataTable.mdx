### useDataTable Hook Documentation

**Description**:
The useDataTable hook is a custom React hook that facilitates filtering and pagination for tabular data. It allows you to manage search queries, pagination, and data filtering efficiently. This hook can be used to enhance the functionality of a data table component in a React application.


## Props    

The `useDataTable` hook accepts the following props:

1. `tableData` (required): An array of objects representing the data for the table. Each object represents a row, and its properties represent columns.

2. `itemsPerPage` (optional, default: 0): An integer representing the number of items to display per page. If set to 0 or not provided, all data will be displayed on a single page without pagination.

3. `queryAll` (optional, default: ""): A string representing a global search query applied to the entire table. This allows users to search for a specific term across all columns.

4. `searchQueryObject` (optional, default: []): An array of search query objects that allow filtering the data based on specific column values. Each object in the array should have two properties:

- `field` (string): The name of the column to filter.
- `value` (string): The value to search for in the specified column.


## Return Value

The `useDataTable` hook returns an object with the following properties and functions:

1. `searchQueries` (array): An array of search query objects currently applied for filtering the data.

2. `setSearchQueries` (function): A function to update the searchQueries. It accepts an array of search query objects.

3. `searchQuery` (string): A global search query applied to the entire table.

4. `setSearchQuery` (function): A function to update the searchQuery. It accepts a string value.

5. `currentPage` (number): The current page number.

6. `setCurrentPage` (function): A function to update the current page. It accepts a numeric value representing the page number.

7. `paginatedData` (array): An array of objects representing the data to be displayed on the current page, after applying search and filtering.

8. `totalItems` (number): The total number of items after applying search and filtering.

9. `totalPages` (number): The total number of pages based on the itemsPerPage and filtered data.

10. `tableHeaders` (array): An array of strings representing the column names of the table.

11. `next` (function): A function to navigate to the next page. It increments the currentPage if there are more pages available.

12. `previous` (function): A function to navigate to the previous page. It decrements the currentPage if the current page is not the first page.

13. `clearFilter` (function): A function to clear all applied filters and search queries. It resets the searchQueries, searchQuery, and currentPage to their initial values.

## Usage

```jsx
import React from "react";
import { useDataTable } from "@locoworks/reusejs-toolkit-react-hooks";";

const YourDataTableComponent = ({ data }) => {
  const {
    searchQueries,
    setSearchQueries,
    searchQuery,
    setSearchQuery,
    currentPage,
    setCurrentPage,
    paginatedData,
    totalItems,
    totalPages,
    tableHeaders,
    next,
    previous,
    clearFilter,
  } = useDataTable({
    tableData: data,
    itemsPerPage: 10,
    queryAll: "",
    searchQueryObject: [
      { field: "column1", value: "" },
      { field: "column2", value: "" },
    ],
  });

  // Your rendering logic for the data table goes here...

  return (
    <div>
      {/* Your data table rendering */}
    </div>
  );
};

export default YourDataTableComponent;

```

**Note:**

Replace `YourDataTableComponent` with the actual component that renders your data table.
Pass the data prop containing your tabular data (array of objects) to the `YourDataTableComponent`.
