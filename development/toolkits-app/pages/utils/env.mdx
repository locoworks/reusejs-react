## Reusejs Env Utility

The `Env` class is a utility class that provides information about the rendering environment and handoff state. These `utility` can work in any js environment

### Properties

- `current` (RenderEnv): The current rendering environment, either "client" or "server".
- `handoffState` (HandoffState): The handoff state, either "pending" or "complete".
- `currentId` (number): The current ID used for generating unique IDs.

### Methods

- `set(env: RenderEnv): void`: Sets the rendering environment to the specified value.
- `reset(): void`: Resets the rendering environment to the detected value.
- `nextId(): number`: Generates the next unique ID.
- `handoff(): void`: Marks the handoff state as complete.

### Computed Properties

- `isServer` (boolean): Indicates if the current rendering environment is server-side.
- `isClient` (boolean): Indicates if the current rendering environment is client-side.
- `isHandoffComplete` (boolean): Indicates if the handoff state is complete.

### Usage

1. Import `env` utility to your react project

```jsx
import { env } from '@locoworks/reusejs-toolkit-utils';
```

2. Following example illustrates how to use env utility:

```typescript
import { env } from "@locoworks/reusejs-toolkit-utils";

// Accessing the rendering environment and handoff state
console.log(env.current); // "client" or "server"
console.log(env.handoffState); // "pending" or "complete"

// Checking the rendering environment
if (env.isServer) {
  // Server-side specific logic
} else {
  // Client-side specific logic
}

// Generating a unique ID
const uniqueId = env.nextId();

// Marking the handoff state as complete
env.handoff();

// Checking if the handoff state is complete
if (env.isHandoffComplete) {
  // Handoff is complete
}

// Setting the rendering environment
env.set("server");

// Resetting the rendering environment
env.reset();
```

The `Env` class provides a way to determine the rendering environment and handoff state in your application. It detects whether the code is running on the server or the client and allows you to perform environment-specific logic. Additionally, it provides the ability to generate unique IDs and manage the handoff state.

The `env` object is an instance of the `Env` class that you can import and use to access the rendering environment and handoff state in your code.
